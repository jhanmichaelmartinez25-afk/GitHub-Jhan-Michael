HTML

HTML son las siglas de Lenguaje de Marcado de Hipertexto (HyperText Markup Language) y es el código estándar que se utiliza para crear y estructurar el contenido de las páginas web. Se compone de etiquetas que le dicen al navegador cómo mostrar elementos como texto, imágenes, videos, enlaces y más, creando así el esqueleto de un sitio web. HTML no es un lenguaje de programación, sino un lenguaje de marcado que define la estructura y disposición del contenido.

Historia

La primera versión de HTML fue escrita por Tim Berners-Lee en 1993. Desde entonces, han existido muchas versiones diferentes. La versión más utilizada durante la década del 2000 fue HTML 4.01 , que se convirtió en estándar oficial en diciembre de 1999. Otra versión, XHTML , fue una reescritura de HTML como lenguaje XML. XML es un lenguaje de marcado estándar que se utiliza para crear otros lenguajes de marcado. Actualmente se utilizan cientos de lenguajes XML, incluidos GML (Geography Markup Language), MathML, MusicML y RSS (Really Simple Syndication). Dado que cada uno de estos lenguajes se escribió en un lenguaje común (XML), su contenido se puede compartir fácilmente entre aplicaciones. Esto hace que XML sea potencialmente muy potente, y no sorprende que el W3C creara una versión XML de HTML (también llamada XHTML). XHTML se convirtió en un estándar oficial en 2000 y se actualizó en 2002. XHTML es muy similar a HTML, pero tiene reglas más estrictas. Las reglas estrictas son necesarias para todos los lenguajes XML, ya que sin ellas, la interoperabilidad entre aplicaciones sería imposible. Aprenderás más sobre las diferencias entre HTML y XHTML en la Unidad 2 . La mayoría de las páginas web actuales se crearon con HTML 4.01 o XHTML 1.0. Sin embargo, en los últimos años, el W3C (en colaboración con WHATWG ) ha estado trabajando en una nueva versión de HTML: HTML5 . Actualmente (2011), HTML5 es aún un borrador de especificación y no un estándar oficial. No obstante, ya cuenta con un amplio soporte en navegadores y otros dispositivos con acceso a internet, y representa el futuro de la tecnología.

Editores de HTML

Los editores de HTML son herramientas de software para crear y editar código HTML, y se dividen principalmente en dos tipos: editores de texto que permiten escribir código directamente y editores WYSIWYG ("lo que ves es lo que obtienes") que ofrecen una interfaz visual para el diseño. Ejemplos populares de editores de texto incluyen Visual Studio Code y Sublime Text, mientras que Adobe Dreamweaver es un ejemplo de editor WYSIWYG. Los atributos HTML

Los atributos HTML son palabras especiales dentro de la etiqueta de apertura de un elemento que proporcionan información adicional, modifican su comportamiento o añaden funcionalidad. Se componen de un nombre y un valor, con la estructura nombre="valor", y siempre se colocan después de la etiqueta de apertura.

Estructura
  • Nombre: Indica el tipo de información adicional (ej. href, src, id).
  • Valor: Es la información específica (ej. la URL en href="https://www.ejemplo.com").
  • Colocación: Siempre dentro de la etiqueta de apertura, después de su nombre.
  • Ejemplo: En la etiqueta , href es el nombre y "https://www.ejemplo.com" es el valor.

Tipos
  • Atributos obligatorios: Necesarios para que un elemento funcione correctamente (ej. alt en la etiqueta para texto alternativo).
  • Atributos opcionales: Modifican el comportamiento predeterminado de un elemento (ej. el atributo class para aplicar estilos).
  • Atributos estándar: Compatibles con muchos tipos de elementos (ej. id o class).
  • Atributos de eventos: Permiten asociar scripts que se ejecutan cuando ocurre un evento específico, como hacer clic (onclick) o pasar el ratón por encima (onmouseover).
  • Atributos globales: Se pueden utilizar en la mayoría de los elementos, como spellcheck (revisión ortográfica) o dir (dirección del texto).

Consistencia y buenas practicas

  1. Minusculas:Aunque HTML no distingue mayúsculas y minúsculas, es una práctica estándar y recomendada usar etiquetas y atributos en minúsculas para mayor consistencia.
  2. Espacios en blanco Los espacios, tabuladores y saltos de línea en el código son ignorados por el navegador en la sección body>
  3. Sangria Usar sangrías para indentar el código facilita la lectura.
  4. ComillasSe recomienda usar comillas dobles (") para los valores de los atributos, incluso si el navegador lo permite sin ellas.

CSS

CSS significa Hojas de Estilo en Cascada (Cascading Style Sheets) y es un lenguaje de estilo utilizado para dar formato y estilo a documentos web escritos en lenguajes como HTML. Separa el contenido de la presentación visual, permitiendo controlar el diseño, los colores, las fuentes, el espaciado y la disposición para que el contenido sea atractivo y adaptable a diferentes dispositivos.
